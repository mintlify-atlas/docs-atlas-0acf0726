## Overview

Kairo's analytics dashboard provides Team Leaders with comprehensive visibility into student performance, soft skills development, and at-risk learners. The system aggregates data from Moodle progress, soft skills assessments, and AI-generated reports.

<Note>
  Dashboards are **clan-scoped**—Team Leaders only see data for students in their assigned clan.
</Note>

## Data Aggregation Logic

### Overview Calculations

<Steps>
  <Step title="Total Coders">
    Count all users with `role = 'coder'` in the TL's clan.
  </Step>

  <Step title="Completed Onboarding">
    Count coders where `first_login IS NULL` (indicates they've logged in at least once after registration).
  </Step>

  <Step title="High Risk Coders">
    Count coders with unresolved risk flags where `risk_level IN ('high', 'critical')`.
    
    ```javascript
    const highRiskCoders = coders.filter(
      (c) => c.risk_level === 'high' || c.risk_level === 'critical'
    ).length;
    ```
  </Step>

  <Step title="Clan Average Score">
    Average of all `moodle_progress.average_score` values for coders in the clan.
    
    ```javascript
    const clanAvgScore = totalCoders > 0
      ? (coders.reduce((acc, c) => acc + parseFloat(c.average_score || 0), 0) / totalCoders).toFixed(1)
      : 0;
    ```
  </Step>
</Steps>

### Soft Skills Averaging

<Warning>
  Only coders who have completed the diagnostic (`learning_style IS NOT NULL`) are included in soft skills averages to avoid skewing results toward zero.
</Warning>

```javascript
const codersWithSkills = coders.filter((c) => c.learning_style !== null);

if (codersWithSkills.length > 0) {
  codersWithSkills.forEach((c) => {
    softSkillsAverage.autonomy += parseFloat(c.autonomy || 0);
    softSkillsAverage.time_management += parseFloat(c.time_management || 0);
    softSkillsAverage.problem_solving += parseFloat(c.problem_solving || 0);
    softSkillsAverage.communication += parseFloat(c.communication || 0);
    softSkillsAverage.teamwork += parseFloat(c.teamwork || 0);
  });

  Object.keys(softSkillsAverage).forEach((key) => {
    softSkillsAverage[key] = parseFloat(
      (softSkillsAverage[key] / codersWithSkills.length).toFixed(1)
    );
  });
}
```

**Implementation:** `backend-node/src/controllers/tlControllers.js:303-329`

## SQL Queries

### Dashboard Main Query

```sql
SELECT 
  u.id, u.full_name, u.email, u.first_login, u.clan AS clan_id,
  mp.average_score, mp.current_week,
  ss.autonomy, ss.time_management, ss.problem_solving, 
  ss.communication, ss.teamwork, ss.learning_style,
  rf.risk_level
FROM users u
LEFT JOIN moodle_progress mp ON u.id = mp.coder_id
LEFT JOIN soft_skills_assessment ss ON u.id = ss.coder_id
LEFT JOIN risk_flags rf ON u.id = rf.coder_id AND rf.resolved = false
WHERE u.role = 'coder' AND u.clan = $1
ORDER BY u.full_name ASC
```

**Note:** Uses `LEFT JOIN` to include coders who haven't started yet or haven't completed diagnostics.

### Metrics Query

```sql
SELECT
  AVG(mp.average_score) AS clan_average,
  COUNT(CASE WHEN rf.risk_level IN ('high','critical') THEN 1 END) AS high_risk_count,
  COUNT(u.id) AS total_coders
FROM users u
LEFT JOIN moodle_progress mp ON u.id = mp.coder_id
LEFT JOIN risk_flags rf ON u.id = rf.coder_id AND rf.resolved = false
WHERE u.clan = $1 AND u.role = 'coder'
```

### PDF Generation Technology

<Note>
  Kairo uses **fpdf2** (pure Python PDF library) to avoid system dependencies like wkhtmltopdf. This ensures portability and eliminates deployment issues.
</Note>

```python
from fpdf import FPDF

def _build_pdf(clan: str, coders: list, skills_map: dict, progress_map: dict) -> bytes:
    pdf = FPDF()
    pdf.set_auto_page_break(auto=True, margin=15)
    pdf.add_page()
    
    # Header with Kairo branding
    pdf.set_font("Helvetica", "B", 20)
    pdf.set_fill_color(109, 40, 217)  # Kairo purple
    pdf.set_text_color(255, 255, 255)
    pdf.cell(0, 14, f"  Clan {clan.upper()} — Performance Report", fill=True, ln=True)
    
    # ... profile sections ...
    
    return bytes(pdf.output())
```

## Performance Optimizations

<Accordion title="Single Query Strategy">
  The dashboard uses one comprehensive LEFT JOIN query instead of multiple round-trips:
  
  ```sql
  SELECT u.*, mp.*, ss.*, rf.risk_level
  FROM users u
  LEFT JOIN moodle_progress mp ON u.id = mp.coder_id
  LEFT JOIN soft_skills_assessment ss ON u.id = ss.coder_id
  LEFT JOIN risk_flags rf ON u.id = rf.coder_id AND rf.resolved = false
  WHERE u.role = 'coder' AND u.clan = $1
  ```
  
  This reduces database load and latency for large clans.
</Accordion>

<Accordion title="Clan-Based Indexing">
  The `users` table should have an index on the `clan` column:
  
  ```sql
  CREATE INDEX idx_users_clan ON users(clan) WHERE role = 'coder';
  ```
</Accordion>

<Accordion title="Materialized Views for Historical Trends">
  For historical analytics, consider materialized views that refresh daily:
  
  ```sql
  CREATE MATERIALIZED VIEW clan_weekly_stats AS
  SELECT 
    u.clan,
    DATE_TRUNC('week', mp.updated_at) AS week_start,
    AVG(mp.average_score) AS avg_score,
    COUNT(*) AS active_coders
  FROM users u
  JOIN moodle_progress mp ON u.id = mp.coder_id
  WHERE u.role = 'coder'
  GROUP BY u.clan, week_start;
  ```
</Accordion>
