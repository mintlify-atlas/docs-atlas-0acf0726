## Overview

Kairo uses AI-powered learning path generation to create personalized study plans tailored to each student's unique profile. The system analyzes soft skills, learning styles, and performance data to generate contextual recommendations.

<Note>
  Learning paths are generated using **Groq's Llama 3.3 70B** model, providing high-quality educational planning with minimal latency.
</Note>

## How It Works

<Steps>
  <Step title="Student Completes Diagnostic">
    The onboarding process collects data through VARK, ILS (Index of Learning Styles), and Kolb assessments to build a comprehensive learner profile.
    
    **Endpoint:** `POST /api/diagnostics`
    
    ```javascript
    {
      "answers": [
        { "questionId": 1, "optionId": "a", "score": "V" },
        { "questionId": 2, "optionId": "b", "score": "ACT" }
      ]
    }
    ```
  </Step>

  <Step title="Soft Skills Derivation">
    The system derives five core soft skill scores from the diagnostic answers:
    
    - **Autonomy** ← REF (reflective) + AC (abstract conceptualization) + RO (reflective observation)
    - **Time Management** ← SEQ (sequential) + AE (active experimentation) + SNS (sensing)
    - **Problem Solving** ← GLO (global) + AC + INT (intuitive)
    - **Communication** ← ACT (active) + CE (concrete experience) + VRB (verbal)
    - **Teamwork** ← ACT + CE + SNS
    
    Each skill is scored 1-5 based on psychological learning patterns.
    
    **Implementation:** `backend-node/src/controllers/diagnosticControllers.js:121-131`
  </Step>

  <Step title="AI Plan Generation">
    Two types of plans are generated:
    
    ### Interpretive Plans
    Generated after onboarding completion. Analyzes soft skills and module structure to create an initial personalized roadmap.
    
    ### Analytical Plans
    Generated weekly based on performance data, struggling topics, and progress metrics. Adapts to the student's evolving needs.
    
    **Endpoint:** `POST /generate-plan` (Python microservice)
  </Step>

  <Step title="Contextual Prompt Building">
    The AI receives rich context including:
    - Student's soft skill profile
    - Learning style (visual, auditory, reading, kinesthetic, mixed)
    - Current module and week data
    - Struggling topics (for analytical plans)
    - Historical performance data
    
    **Implementation:** `backend-python/app/services/prompt_builder.py`
  </Step>

  <Step title="Structured Plan Delivery">
    The LLM returns a JSON-structured plan with:
    - Targeted soft skill to develop
    - Learning style adaptation strategies
    - Week-by-week activities and resources
    - Personalized recommendations
    
    Plans are stored in `complementary_plans` table with versioning support.
  </Step>
</Steps>

### Groq Client Setup

The system uses Groq's API for fast inference with OpenAI-compatible SDK:

```python
# backend-python/app/services/ia_services.py
from groq import Groq

client = Groq(api_key=os.getenv("GROQ_API_KEY"))

response = client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    messages=[
        {
            "role": "system",
            "content": (
                "Eres Kairo, un arquitecto educativo de Riwi. "
                "Respondes ÚNICAMENTE con JSON válido, sin texto adicional."
            )
        },
        {"role": "user", "content": prompt}
    ],
    temperature=0.7,
    max_tokens=8192,
    response_format={"type": "json_object"}
)
```

## Data Models

### Soft Skills Assessment

```sql
CREATE TABLE soft_skills_assessment (
  coder_id         INT PRIMARY KEY,
  autonomy         INT CHECK (autonomy BETWEEN 1 AND 5),
  time_management  INT CHECK (time_management BETWEEN 1 AND 5),
  problem_solving  INT CHECK (problem_solving BETWEEN 1 AND 5),
  communication    INT CHECK (communication BETWEEN 1 AND 5),
  teamwork         INT CHECK (teamwork BETWEEN 1 AND 5),
```


## Background Plan Generation

Plans are generated asynchronously to avoid blocking the user experience:

```javascript
// Fire-and-forget pattern
async function _triggerInterpretivePlan(coderId, moduleId, currentWeek = 1) {
  try {
    const res = await fetch(`${PYTHON_API}/generate-plan`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        coder_id: coderId,
        module_id: moduleId,
        plan_type: 'interpretive',
        current_week: currentWeek
      })
    });
    // Non-blocking - frontend receives immediate response
  } catch (err) {
    // Never crash the main flow
    console.error('[Plan Gen] Failed:', err.message);
  }
}
```

**Implementation:** `backend-node/src/controllers/diagnosticControllers.js:34-60`

## Plan Versioning

When a new plan is generated, the system:

1. Deactivates all previous plans for the coder
2. Inserts the new plan with `is_active = true`
3. Preserves historical plans for analytics

```javascript
// Transaction-based plan storage
await client.query('BEGIN');
await client.query(
  'UPDATE complementary_plans SET is_active = false WHERE coder_id = $1',
  [coderId]
);
await client.query(
  `INSERT INTO complementary_plans 
   (coder_id, module_id, plan_content, targeted_soft_skill, is_active) 
   VALUES ($1, $2, $3, $4, true)`,
  [coderId, moduleId, planContent, targetedSoftSkill]
);
await client.query('COMMIT');
```
