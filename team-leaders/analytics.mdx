## Risk Flag System

### How Risk Levels Work

Kairo's AI system automatically flags students based on multiple signals:

<Steps>
  <Step title="Performance Monitoring">
    Tracks Moodle average scores, completion rates, and weekly progress
  </Step>
  <Step title="Soft Skills Analysis">
    Identifies low scores in autonomy, time management, or problem-solving
  </Step>
  <Step title="Engagement Patterns">
    Detects irregular login patterns or declining activity
  </Step>
  <Step title="AI Risk Classification">
    Assigns risk level: **low**, **medium**, **high**, or **critical**
  </Step>
</Steps>

### Risk Levels Explained

| Level | Criteria | Recommended Action |
|-------|----------|--------------------|
| **Critical** | Score < 50 AND low soft skills | Immediate 1:1 intervention required |
| **High** | Score < 60 OR 2+ failing soft skills | Schedule feedback session this week |
| **Medium** | Score 60-70 OR 1 low soft skill | Monitor closely, provide resources |
| **Low** | Score > 70 AND healthy soft skills | Continue standard support |

<Accordion title="Backend Implementation">
  Risk flags are stored in the `risk_flags` table with the following schema:

  ```sql
  CREATE TABLE risk_flags (
    id SERIAL PRIMARY KEY,
    coder_id INT REFERENCES users(id),
    risk_level VARCHAR(20), -- 'low', 'medium', 'high', 'critical'
    reason TEXT,
    resolved BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
  );
  ```

  Query to fetch unresolved risks (tlControllers.js:224):
  ```javascript
  SELECT rf.*, u.full_name, u.email
  FROM risk_flags rf
  JOIN users u ON rf.coder_id = u.id
  WHERE rf.resolved = false
  ORDER BY
    CASE rf.risk_level
      WHEN 'critical' THEN 1
      WHEN 'high' THEN 2
      WHEN 'medium' THEN 3
      ELSE 4
    END
  ```
</Accordion>

## Struggling Topics Analysis

The system tracks which topics students are struggling with based on Moodle activity performance.

### Data Source

**Database Column:** `moodle_progress.struggling_topics` (array of topic IDs)

### Identifying Common Struggles

<Steps>
  <Step title="Aggregate Topic IDs">
    Collect struggling_topics arrays from all students in your clan
  </Step>
  <Step title="Count Frequency">
    Determine which topics appear most often across students
  </Step>
  <Step title="Map to Topic Names">
    Query the `topics` table to get human-readable names
  </Step>
  <Step title="Prioritize Interventions">
    Focus group coaching sessions on the most common struggles
  </Step>
</Steps>

<Accordion title="Example Query Pattern">
  ```javascript
  // Get struggling topics for all clan members
  const { data: progress } = await supabase
    .from('moodle_progress')
    .select('struggling_topics')
    .in('coder_id', coderIds);

  // Flatten and count topic IDs
  const topicCounts = {};
  progress.forEach(p => {
    (p.struggling_topics || []).forEach(topicId => {
      topicCounts[topicId] = (topicCounts[topicId] || 0) + 1;
    });
  });

  // Fetch topic names for top 5 most common
  const topTopicIds = Object.keys(topicCounts)
    .sort((a, b) => topicCounts[b] - topicCounts[a])
    .slice(0, 5);

  const { data: topics } = await supabase
    .from('topics')
    .select('id, name')
    .in('id', topTopicIds);
  ```
</Accordion>

## AI-Powered Insights

### Real-Time Risk Detection

The AI system continuously monitors student behavior and flags anomalies:

- **Sudden score drops** (>15 points in one week)
- **Prolonged inactivity** (no login for 48+ hours)
- **Low engagement** (minimal time in Moodle activities)
- **Soft skill red flags** (autonomy or problem-solving < 2.5)

### Predictive Analytics (Coming Soon)

Upcoming features include:
- **Dropout risk prediction** - Identify students likely to leave the program
- **Performance forecasting** - Estimate final scores based on current trajectory
- **Optimal intervention timing** - AI recommends when to reach out to each student

## Best Practices for Data-Driven Leadership

<Steps>
  <Step title="Weekly Analytics Review">
    Every Monday, pull clan metrics to set the week's intervention priorities
  </Step>
  <Step title="Identify Patterns">
    Look for correlations - e.g., "students with low autonomy struggle with async modules"
  </Step>
  <Step title="Segment by Risk Level">
    Create separate engagement strategies for critical, high, medium, and low-risk students
  </Step>
  <Step title="Track Intervention Impact">
    After providing feedback, monitor if the student's scores/skills improve
  </Step>
  <Step title="Share Insights">
    Communicate trends to other TLs and instructors for collaborative problem-solving
  </Step>
</Steps>
