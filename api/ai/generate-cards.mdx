## Overview

Generates 6 smart study cards tailored to a student's learning profile and struggling topics. Cards are prioritized and include theory explanations, practical exercises, and pro tips.

Uses Groq's `llama-3.3-70b-versatile` model to create personalized learning materials.

## Authentication

Requires authentication with `coder` role.

```http
Authorization: Bearer <token>
```

## Request Body

<ParamField body="coder_id" type="integer" required>
  The unique identifier of the student
</ParamField>

<ParamField body="module_id" type="integer" required>
  The module context for card generation
</ParamField>

<ParamField body="struggling_topics" type="array" default="[]">
  List of topic names where student needs help
  ```json
  ["Variables", "Functions", "Loops"]
  ```
  If empty, generates cards on "general programming concepts"
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if cards were generated successfully
</ResponseField>

<ResponseField name="data" type="object">
  Container for the generated cards
  
  <ResponseField name="data.cards" type="array">
    Array of 6 focus cards (2 High, 2 Medium, 2 Low priority)
    
    <ResponseField name="cards[].priority" type="string">
      Priority level: `high`, `medium`, or `low`
    </ResponseField>
    
    <ResponseField name="cards[].topic" type="string">
      Topic name (e.g., "Variables", "Functions")
    </ResponseField>
    
    <ResponseField name="cards[].title" type="string">
      Card title (concise learning objective)
    </ResponseField>
    
    <ResponseField name="cards[].theory" type="string">
      3-4 sentence explanation of the concept
    </ResponseField>
    
    <ResponseField name="cards[].practice" type="object">
      Practical exercise
      
      <ResponseField name="practice.description" type="string">
        What the student should do
      </ResponseField>
      
      <ResponseField name="practice.example" type="string">
        Code snippet or step-by-step instructions
      </ResponseField>
    </ResponseField>
    
    <ResponseField name="cards[].pro_tips" type="array">
      Array of 2 actionable tips
      ```json
      ["Tip 1", "Tip 2"]
      ```
    </ResponseField>
  </ResponseField>
</ResponseField>

## AI Model Configuration

**Provider**: Groq  
**Model**: `llama-3.3-70b-versatile`  
**Temperature**: 0.7  
**Max Tokens**: 2048  
**System Prompt**: "You are an expert coding tutor. Respond only with valid JSON, no markdown."

## Context Used

The AI considers:
- **Learning Style**: From soft skills assessment (visual, kinesthetic, mixed)
- **Problem Solving**: Student's problem-solving score (1-5)
- **Autonomy**: Student's autonomy level (1-5)
- **Struggling Topics**: Current challenges from Moodle progress

## Card Distribution

Always generates exactly **6 cards**:
- **2 High Priority**: Core concepts student must master immediately
- **2 Medium Priority**: Important supporting concepts
- **2 Low Priority**: Enhancement topics or best practices

If fewer than 6 cards are returned, a warning is logged.

## Error Handling

<ResponseField name="error" type="object">
  Error details when generation fails
  
  <ResponseField name="status_code" type="integer">
    HTTP status code
  </ResponseField>
  
  <ResponseField name="detail" type="string">
    Error message
  </ResponseField>
</ResponseField>

**Common Errors**:
- `500`: JSON parse error (AI returned invalid JSON)
- `500`: AI generation failed
- `502`: AI service unavailable

## JSON Cleanup

The endpoint automatically handles:
- Markdown code fences (```json)
- Extra whitespace
- Accidental markdown formatting

This ensures reliable parsing even when the LLM includes formatting artifacts.

## Integration with Soft Skills

Cards adapt to student profile:

| Learning Style | Card Adaptation |
|----------------|----------------|
| Visual | Includes diagrams, flowcharts in examples |
| Kinesthetic | Emphasizes hands-on coding exercises |
| Mixed | Balanced theory and practice |

| Problem Solving | Impact |
|-----------------|--------|
| Low (1-2) | More scaffolded steps, detailed examples |
| High (4-5) | Challenging exercises, less guidance |

## Performance

Typical generation time: **1-2 seconds**  
Model inference: **~800ms**  
JSON parsing: **~50ms**
