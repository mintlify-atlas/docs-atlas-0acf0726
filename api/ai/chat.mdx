## Overview

Provides an interactive AI tutor that answers student programming questions in a concise, motivating, and practical way.

Powered by Groq's `llama-3.3-70b-versatile` model, optimized for quick educational responses.

## Authentication

Requires authentication. Can be accessed by any authenticated user.

```http
Authorization: Bearer <token>
```

## Request Body

<ParamField body="message" type="string" required>
  The student's question or message to the AI tutor
</ParamField>

<ParamField body="coder_id" type="integer">
  Optional student ID for personalized context (future use)
</ParamField>

## Response

<ResponseField name="reply" type="string">
  The AI tutor's response (max 3 paragraphs, ~600 tokens)
</ResponseField>

## AI Model Configuration

**Provider**: Groq  
**Model**: `llama-3.3-70b-versatile`  
**Temperature**: 0.7  
**Max Tokens**: 600  
**Response Style**: Concise, motivating, practical

## Response Structure

Kairo follows a **3-part answer structure**:

1. **Short Direct Answer** (1-2 sentences)  
   Immediately addresses the core question

2. **Explanation with Example** (1 paragraph)  
   Provides context with practical code snippet

3. **Best Practice + Motivation** (1-2 sentences)  
   Actionable tip and encouraging closing

## Example Interactions

### Basic Concept Question

**Student**: "What is a function?"

**Kairo**: 
```
¡Gran pregunta! Una función es un bloque de código reutilizable 
que hace una tarea específica.

Piensa en una función como una máquina: le das algo (parámetros), 
hace su trabajo, y te devuelve un resultado. Por ejemplo:

function saludar(nombre) {
  return "Hola " + nombre;
}

console.log(saludar("Ana")); // "Hola Ana"

Las funciones te ahorran escribir el mismo código muchas veces. 
¡Practica creando funciones pequeñas primero!
```

### Debugging Help

**Student**: "My loop is not working, it prints nothing"

**Kairo**:
```
Probablemente tu condición inicial ya es falsa, o no estás 
inicializando bien la variable.

Revisa estos 3 puntos:
1. ¿Declaraste la variable antes del bucle?
2. ¿La condición permite al menos una iteración?
3. ¿Estás incrementando/decrementando correctamente?

Ejemplo común de error:
for (let i = 10; i < 5; i++) { ... } // ¡Nunca ejecuta!

Prueba con console.log(i) antes del bucle para verificar el valor inicial. 
¡El debugging es una habilidad clave, sigue practicando!
```

### Conceptual Understanding

**Student**: "When should I use async/await?"

**Kairo**:
```
Usa async/await cuando necesites esperar operaciones que toman tiempo, 
como llamadas a APIs, lectura de archivos, o consultas a bases de datos.

Es mucho más legible que usar .then(). Ejemplo:

// Con async/await (más claro)
const data = await fetch('/api/users');
const json = await data.json();

// Vs. .then() (más complejo)
fetch('/api/users').then(r => r.json()).then(json => {...});

Recuerda que solo puedes usar `await` dentro de funciones marcadas 
como `async`. ¡Practica con fetch primero!
```

## Response Characteristics

✅ **Always includes**:
- Direct answer first
- Practical code example
- Motivational closing
- Simple language (avoid jargon)

❌ **Never includes**:
- Overly technical explanations
- Long theoretical discussions
- Complete solutions to homework
- More than 3 paragraphs

- **Per IP**: 100 requests/minute

## Use Cases

```javascript
const sendMessage = async (userMessage) => {

  const response = await fetch('/chat/ask', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },

  });
  
  const { reply } = await response.json();
  
  setChatHistory([...chatHistory, 
    { role: 'user', content: userMessage },
    { role: 'assistant', content: reply }
  ]);
  
  setLoading(false);
};
```

### Embedded Help System

```javascript
// Contextual help in code editor
const getHelp = async (selectedCode, errorMessage) => {
  const message = `I'm getting this error: ${errorMessage}\n\nMy code:\n${selectedCode}`;
  
  const response = await fetch('/chat/ask', {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${token}` },
    body: JSON.stringify({ message })
  });
  
  const { reply } = await response.json();
  showHelpPanel(reply);
};
```

## Performance

Typical response time: **400-800ms**
- Model inference: **~300ms**
- Network latency: **~100-200ms**
- API overhead: **~100ms**

## Best Practices

### For Students
- Ask specific questions rather than "How do I code?"
- Include error messages when debugging
- Share relevant code snippets
- Try to solve first, then ask for guidance

### For Integration
- Show loading state while waiting for response
- Display responses with proper markdown rendering
- Implement retry logic for network failures
- Cache common questions/answers locally

## Accessibility

The chat endpoint can be accessed:
- **Directly**: From frontend chat widget
- **Via Node.js**: Through `/api/ia/chat` proxy route (if configured)
- **Mobile**: Native apps via REST API
