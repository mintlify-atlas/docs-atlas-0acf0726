## Overview

Generates AI-powered personalized learning plans for students. Supports two plan types:
- **Interpretive**: Initial plan based on soft skills diagnostic and module structure
- **Analytical**: Adaptive plan that incorporates weekly performance data and struggling topics

The endpoint uses Groq's `llama-3.3-70b-versatile` model to generate customized weekly learning paths.

## Authentication

Requires authentication with `coder` role.

```http
Authorization: Bearer <token>
```

## Request Body

<ParamField body="coder_id" type="integer" required>
  The unique identifier of the student
</ParamField>

<ParamField body="module_id" type="integer" required>
  The module for which to generate the plan
</ParamField>

<ParamField body="plan_type" type="string" default="interpretive">
  Type of plan to generate. Options:
  - `interpretive`: Initial plan based on soft skills
  - `analytical`: Performance-based adaptive plan
</ParamField>

<ParamField body="current_week" type="integer" default="1">
  Current week number (required for analytical plans)
</ParamField>

<ParamField body="average_score" type="number" default="0.0">
  Student's average score (required for analytical plans)
</ParamField>

<ParamField body="struggling_topics" type="array">
  List of topic names where student is struggling (for analytical plans)
  ```json
  ["Variables", "Functions", "Loops"]
  ```
</ParamField>

<ParamField body="weeks_completed" type="array">
  Historical weekly performance data (for analytical plans)
  ```json
  [
    {
      "week": 1,
      "average_score": 85.5,
      "struggling_topics": ["Variables"]
    }
  ]
  ```
</ParamField>

**Temperature**: 0.7

**Response Format**: JSON object

## Workflow

1. **Validation**: Validates `plan_type` is either `interpretive` or `analytical`
2. **Data Retrieval**: Fetches soft skills diagnostic, module details, and weeks from Supabase
3. **Deactivation**: Deactivates any existing active plan for the student
4. **Context Building**: Constructs context with student profile and performance data
5. **AI Generation**: Calls Groq LLM with appropriate prompt based on plan type
6. **Persistence**: Saves plan and logs generation metadata

## Error Handling

<ResponseField name="error" type="object">
  Error details when generation fails
  
  <ResponseField name="status_code" type="integer">
    HTTP status code
  </ResponseField>
  
  <ResponseField name="detail" type="string">
    Error message
  </ResponseField>
</ResponseField>

**Common Errors**:
- `400`: Invalid `plan_type` or missing soft skills diagnostic
- `404`: Module not found
- `502`: AI service unavailable
- `500`: Internal server error

## Fallback Behavior

If Groq API fails, the system returns a safe fallback plan:

```json
{
  "plan_type": "interpretive",
  "status": "fallback",
  "targeted_soft_skill": "autonomy",
  "learning_style_applied": "mixed",
  "summary": "Plan no disponible temporalmente: [reason]",
  "weeks": []
}
```

## Performance

Typical generation time: **1.5-3 seconds**  
Logged in `ai_generation_log` table with `execution_time_ms`
