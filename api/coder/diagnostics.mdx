## Authentication

All diagnostic endpoints require:
- **Session-based authentication** via `req.session.userId`
- **Role:** `coder`

Diagnostic assessment is typically completed during onboarding before dashboard access is granted.

## Soft Skills Assessment

The Kairo platform uses a comprehensive soft skills diagnostic to personalize learning plans. Assessment data is stored in the `soft_skills_assessment` table with the following structure:

### Assessment Fields

<ParamField body="autonomy" type="number" required>
  Autonomy score (1-5 scale)
</ParamField>

<ParamField body="timeManagement" type="number" required>
  Time management score (1-5 scale)
</ParamField>

<ParamField body="problemSolving" type="number" required>
  Problem-solving score (1-5 scale)
</ParamField>

<ParamField body="communication" type="number" required>
  Communication score (1-5 scale)
</ParamField>

<ParamField body="teamwork" type="number" required>
  Teamwork score (1-5 scale)
</ParamField>

<ParamField body="learningStyle" type="string" required>
  Identified learning style (e.g., `"visual"`, `"auditory"`, `"kinesthetic"`, `"reading_writing"`)
</ParamField>

<ParamField body="rawAnswers" type="array" optional>
  Array of raw diagnostic question answers for audit/re-analysis purposes. This field is persisted as JSONB in the database.
</ParamField>

## Partial Update

The soft skills model supports partial updates, though this is typically handled internally by the platform rather than exposed as a direct API endpoint.

### Database Model Methods

**Location:** `backend-node/src/models/softSkills.js:71-96`

The `update(coderId, updates)` function allows updating specific fields without overwriting the entire assessment:

```javascript Example Usage (Internal)
import { update } from '../models/softSkills.js';

// Update only time management score
const updated = await update(coderId, {
  timeManagement: 4
});
```

This method:
- Only updates provided fields
- Automatically updates `assessed_at` timestamp
- Returns the complete updated record
- Returns `null` if no fields provided

## Data Persistence

All soft skills data is stored in the `soft_skills_assessment` table with the following characteristics:

- **Upsert behavior:** `ON CONFLICT (coder_id) DO UPDATE` ensures one assessment per student
- **JSONB storage:** `raw_answers` column stores full diagnostic responses for audit/re-analysis
- **Auto-timestamps:** `assessed_at` updates automatically on every upsert
- **Unique constraint:** One assessment per `coder_id`

### Example Database Record

```sql
SELECT * FROM soft_skills_assessment WHERE coder_id = 42;
```
