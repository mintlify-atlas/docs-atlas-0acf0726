## Get Clan Students

<RequestExample>
```bash cURL
curl -X GET https://api.kairolearning.com/api/tl/coders/clan/clan-alpha \
  -H "Cookie: session_id=your_session_token"
```

```javascript JavaScript
const clanId = 'clan-alpha';
const response = await fetch(`/api/tl/coders/clan/${clanId}`, {
  credentials: 'include'
});
const data = await response.json();
```

```python Python
import requests

clan_id = 'clan-alpha'
response = requests.get(
    f'https://api.kairolearning.com/api/tl/coders/clan/{clan_id}',
    cookies={'session_id': 'your_session_token'}

data = response.json()
```

### Endpoint

```
GET /api/tl/coders/clan/:clan_id
```

### Description

Retrieves all students (coders) in a specified clan with their latest progress metrics and unresolved risk flags. Results are ordered by risk severity (critical → high → medium → none) and then alphabetically by name.

### Path Parameters

<ParamField path="clan_id" type="string" required>
  The clan identifier (e.g., `clan-alpha`, `clan-bravo`)
</ParamField>

### Authentication

Requires active session with `tl` (Team Leader) role.

### Response Fields

<ResponseField name="clan" type="string">
  The clan ID from the request
</ResponseField>

<ResponseField name="total" type="integer">
  Total number of students in the clan
</ResponseField>

<ResponseField name="coders" type="array">
  Array of student records with risk prioritization
  
  <ResponseField name="id" type="integer">
    Student user ID
  </ResponseField>
  
  <ResponseField name="full_name" type="string">
    Student's full name
  </ResponseField>
  
  <ResponseField name="email" type="string">
    Student's email address
  </ResponseField>
  
  <ResponseField name="average_score" type="number | null">
    Average Moodle quiz score across completed weeks
  </ResponseField>
  
  <ResponseField name="current_week" type="integer | null">
    Current week of course progression
  </ResponseField>
  
  <ResponseField name="risk_level" type="string | null">
    Active risk flag level: `critical`, `high`, `medium`, or `null`
  </ResponseField>
  
  <ResponseField name="risk_reason" type="string | null">
    Explanation of why the risk flag was raised
  </ResponseField>
</ResponseField>

### Error Responses

<ResponseField name="500" type="object">
  Server error during data retrieval
  ```json
  { "error": "Failed to fetch clan data" }
  ```
</ResponseField>

## Submit Feedback

<RequestExample>
```bash cURL
curl -X POST https://api.kairolearning.com/api/tl/feedback \
  -H "Content-Type: application/json" \
  -H "Cookie: session_id=your_session_token" \
  -d '{
    "coderId": 42,
    "feedbackText": "Great progress on the React module! Focus on improving error handling in your API calls.",
    "feedbackType": "technical"
  }'
```

```javascript JavaScript
const response = await fetch('/api/tl/feedback', {
  method: 'POST',
  credentials: 'include',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    coderId: 42,
    feedbackText: 'Great progress on the React module! Focus on improving error handling in your API calls.',
    feedbackType: 'technical'
  })
});
const data = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://api.kairolearning.com/api/tl/feedback',
    cookies={'session_id': 'your_session_token'},
    json={
        'coderId': 42,
        'feedbackText': 'Great progress on the React module! Focus on improving error handling in your API calls.',
        'feedbackType': 'technical'
    }
)
data = response.json()
```
</RequestExample>

<ResponseExample>
```json Response
{
  "success": true,
  "message": "Feedback registered",
  "data": {
    "id": 128,
    "coder_id": 42,
    "tl_id": 5,
    "feedback_text": "Great progress on the React module! Focus on improving error handling in your API calls.",
    "feedback_type": "technical",
    "created_at": "2026-03-14T15:45:00.000Z"
  }
}
```
</ResponseExample>

### Endpoint

```
POST /api/tl/feedback
```

### Description

Submits feedback from a Team Leader to a student. Automatically sends a real-time notification to the student.

### Authentication

Requires active session with `tl` (Team Leader) role. The TL ID is extracted from the session.

### Request Body

<ParamField body="coderId" type="integer" required>
  The student's user ID
</ParamField>

<ParamField body="feedbackText" type="string" required>
  The feedback message content (sanitized on server)
</ParamField>

<ParamField body="feedbackType" type="string" required>
  Category of feedback (e.g., `technical`, `behavioral`, `progress`, `soft_skills`)
</ParamField>

### Response Fields

<ResponseField name="success" type="boolean">
  Indicates if the feedback was successfully registered
</ResponseField>

<ResponseField name="message" type="string">
  Status message
</ResponseField>

<ResponseField name="data" type="object">
  The created feedback record
  
  <ResponseField name="id" type="integer">
    Feedback record ID
  </ResponseField>
  
  <ResponseField name="coder_id" type="integer">
    Student user ID
  </ResponseField>
  
  <ResponseField name="tl_id" type="integer">
    Team Leader user ID
  </ResponseField>
  
  <ResponseField name="feedback_text" type="string">
    Sanitized feedback text
  </ResponseField>
  
  <ResponseField name="feedback_type" type="string">
    Sanitized feedback type
  </ResponseField>
  
  <ResponseField name="created_at" type="string">
    ISO 8601 timestamp of feedback submission
  </ResponseField>
</ResponseField>

### Error Responses

<ResponseField name="400" type="object">
  Missing required fields
  ```json
  { "error": "All fields are required" }
  ```
</ResponseField>

<ResponseField name="500" type="object">
  Server error during feedback submission
  ```json
  { "error": "Failed to submit feedback" }
  ```
</ResponseField>

### Side Effects

- Creates a record in the `tl_feedback` table
- Sends a real-time notification to the student via `notifyUser()` service
- Notification preview shows first 50 characters of feedback text
