## Welcome to Kairo

Kairo is an AI-powered learning platform designed to personalize the educational experience for Riwi bootcamp students. This guide will walk you through setting up your account and generating your first personalized learning plan.

<Note>
  **What You'll Accomplish:**
  - Create and verify your account
  - Complete the onboarding diagnostic
  - Generate your first AI-powered learning plan
  - Understand your personalized learning path
</Note>

---

## Prerequisites

Before you begin, ensure you have:

- A valid email address
- Modern web browser (Chrome, Firefox, Safari, or Edge)
- Access to your email for OTP verification

## Getting Started

<Steps>
  <Step title="Create Your Account">
    Navigate to the registration page and provide your details:

    ```bash
    POST /api/auth/register
    ```

    **Request Body:**
    ```json
    {
      "email": "student@riwi.io",
      "password": "SecurePass123!",
      "fullName": "Maria Garcia",
      "role": "coder",
      "clanId": 1
    }
    ```

    **Response:**
    ```json
    {
      "message": "Código enviado. Verifica tu correo para completar el registro.",
      "email": "student@riwi.io"
    }
    ```

    <Note>
      The system uses OTP (One-Time Password) verification for security. Check your email inbox for the 6-digit code.
    </Note>
  </Step>

  <Step title="Verify Your Email">
    Enter the 6-digit code sent to your email:

    ```bash
    POST /api/auth/verify-otp
    ```

    **Request Body:**
    ```json
    {
      "email": "student@riwi.io",
      "code": "123456"
    }
    ```

    **Success Response:**
    ```json
    {
      "success": true,
      "user": {
        "id": 42,
        "email": "student@riwi.io",
        "fullName": "Maria Garcia",
        "role": "coder",
        "clanId": 1,
        "firstLogin": true
      }
    }
    ```

    <Warning>
      OTP codes expire after 15 minutes. If your code expires, use the "Resend OTP" option.
    </Warning>
  </Step>

  <Step title="Complete the Onboarding Diagnostic">
    New users will be redirected to the onboarding flow. This diagnostic assesses your soft skills to personalize your learning experience.

    **Soft Skills Evaluated:**
    - Emotional Intelligence
    - Communication Skills
    - Critical Thinking
    - Adaptability
    - Teamwork

    The diagnostic uses a 5-point scale for each skill area and takes approximately 5-10 minutes to complete.

    <Tip>
      Answer honestly! The AI uses this data to create learning plans that strengthen your unique skill gaps.
    </Tip>
  </Step>

  <Step title="Generate Your First Learning Plan">
    Once onboarding is complete, navigate to your dashboard and click "Generate Learning Plan".

    ```bash
    POST /api/ai/generate-plan
    ```

    **The AI analyzes:**
    - Your current module progress from Moodle
    - Soft skills assessment results
    - Topics you're struggling with
    - Your learning pace

    **Sample Response:**
    ```json
    {
      "success": true,
      "plan_id": 789,
      "plan": {
        "title": "Personalized Growth Plan - Week 1-4",
        "targeted_soft_skill": "critical_thinking",
        "weeks": [
          {
            "week": 1,
            "focus": "JavaScript Fundamentals",
            "activities": [
              {
                "type": "study",
                "title": "Variables & Data Types",
                "duration": "2 hours",
                "description": "Master the basics of JavaScript variables..."
              },
              {
                "type": "practice",
                "title": "Code Challenge: Array Methods",
                "duration": "1 hour",
                "description": "Apply your knowledge to solve real problems..."
              }
            ]
          }
        ]
      },
      "metadata": {
        "plan_type": "interpretive",
        "module_name": "Frontend Development",
        "current_week": 1,
        "execution_ms": 3421
      }
    }
    ```
  </Step>

  <Step title="Start Learning">
    Your personalized plan is now active! Access it from your dashboard to:

    - View weekly learning objectives
    - Complete recommended exercises
    - Track your progress
    - Generate focus cards for struggling topics

    <Card title="Pro Tip" icon="lightbulb">
      The platform generates two types of plans:
      - **Interpretive Plans**: Created at the start of a module based on your diagnostic
      - **Analytical Plans**: Generated weekly based on your actual performance data
    </Card>
  </Step>
</Steps>

## Common Workflows

### Weekly Progress Update

The platform automatically tracks your Moodle performance. Each Monday:

1. System fetches your latest scores and struggling topics
2. AI generates an **analytical plan** based on actual data
3. Previous plan is deactivated (accessible in history)
4. New plan becomes active on your dashboard

### Requesting Help

If you're struggling with a topic:

1. Navigate to **Focus Cards** section
2. Click **Generate Cards** for struggling topics
3. Receive AI-generated study materials with:
   - Concept explanations
   - Code examples
   - Practice exercises
   - Additional resources

### Tracking Progress

View your learning analytics:
- Moodle score trends
- Soft skills development
- Completed vs. pending activities
- Time spent on each topic

## Session Management

Kairo uses secure session-based authentication:

```javascript
// Session Configuration (backend-node/src/server.js:59-72)
app.use(
  session({
    name: 'riwi.sid',
    secret: process.env.SESSION_SECRET,
    resave: false,
    saveUninitialized: false,
    cookie: {
      secure: isProduction,      // HTTPS only in production
      httpOnly: true,             // Prevent XSS attacks
      maxAge: 24 * 60 * 60 * 1000, // 24 hours
      sameSite: isProduction ? 'none' : 'lax'
    }
  })
);
```

**Logout:**
```bash
POST /api/auth/logout
```

Response:
```json
{
  "message": "Sesión cerrada."
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="OTP Code Not Received">
    **Solutions:**
    1. Check your spam/junk folder
    2. Verify the email address is correct
    3. Click "Resend OTP" (max 3 times per 10 minutes)
    4. Wait 5 minutes and try again

    The OTP system uses Nodemailer with Gmail SMTP:
    ```javascript
    // backend-node/src/services/email.service.js
    SMTP_HOST=smtp.gmail.com
    SMTP_PORT=465
    SMTP_SECURE=true
    ```
  </Accordion>

  <Accordion title="Session Expires Too Quickly">
    Sessions last 24 hours. If you're experiencing premature logouts:
    
    1. Ensure cookies are enabled in your browser
    2. Check if you're using private/incognito mode
    3. Verify your system time is correct
    4. Clear browser cache and try again
  </Accordion>

  <Accordion title="Learning Plan Not Generating">
    Plans require:
    1. Completed onboarding diagnostic
    2. Active Moodle enrollment
    3. Python AI service running

    Check AI service health:
    ```bash
    GET /api/ai/health
    ```

    Expected response:
    ```json
    {
      "node": "ok",
      "python": {
        "status": "online",
        "service": "Kairo AI Engine",
        "model": "llama-3.3-70b-versatile",
        "embeddings": "all-MiniLM-L6-v2"
      }
    }
    ```
  </Accordion>

  <Accordion title="Can't Access Dashboard After Login">
    Ensure you completed the onboarding flow. First-time users with `firstLogin: true` are automatically redirected to `/onboarding.html`.

    Update your onboarding status:
    ```bash
    PATCH /api/auth/complete-onboarding
    ```

    Request body:
    ```json
    {
      "clanId": 1
    }
    ```
  </Accordion>
</AccordionGroup>
